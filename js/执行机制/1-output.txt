1.js 代码执行结果说明
================================

实际输出：
-----------
[Function: a]
2
1


详细执行过程分析：
-----------

第1个输出：[Function: a]
  来自 fn(3) 函数内第一个 console.log(a)
  此时 a 在创建阶段被函数声明覆盖，值为 function a() {}

第2个输出：2
  来自 fn(3) 函数内第二个 console.log(a)
  此时执行了 var a = 2，a 被赋值为数字 2

第3个输出：1
  来自全局的 console.log(a)
  全局变量 a 一直保持为 1，不受函数内部影响


变量 a 的值变化时间线：
-----------

时间点0：全局执行上下文创建
  全局 a = undefined → 1

时间点1：fn(3) 调用，创建函数执行上下文（创建阶段）
  步骤1：处理参数        → a = 3
  步骤2：处理函数声明     → a = function a() {}  (覆盖参数)
  步骤3：处理var声明      → a已存在，跳过；b = undefined

时间点2：执行阶段 - console.log(a)
  输出：function a() {}
  此时 a = function a() {}

时间点3：执行 var a = 2
  a = 2  (赋值操作)
  此时 a = 2

时间点4：执行 function a() {}
  无操作（函数声明已在创建阶段处理）
  此时 a = 2 (不变)

时间点5：执行 var b = a
  b = 2
  此时 a = 2, b = 2

时间点6：执行 console.log(a)
  输出：2
  此时 a = 2

时间点7：函数执行完毕，返回全局上下文
  执行 console.log(a)
  输出：1
  全局 a = 1


核心知识点：
-----------

1. 创建阶段的处理顺序：
   参数 → 函数声明 → var声明
   
2. 优先级规则：
   函数声明 > 参数 > var声明
   
3. 执行阶段：
   - 函数声明被跳过（已提升）
   - var声明被跳过（已提升）
   - 只执行赋值操作
   
4. 作用域隔离：
   函数内的局部变量不影响外部作用域


如何运行代码：
-----------

方法1：使用Node.js
  node js/执行机制/1.js

方法2：在浏览器控制台
  复制1.js的代码到浏览器开发者工具的Console中运行

方法3：在VSCode中
  右键 1.js → Run Code (需要Code Runner插件)
