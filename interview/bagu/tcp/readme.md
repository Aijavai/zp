- 三次握手 
  - 建立连接
  - A 请求 SEQ  j 
  - B 响应 ACK j + 1 
  - A 确认 ACK  B 的 SEQ + 1
  - B 确认 ACK A 的 SEQ + 1

  发送数据以及接收数据的能力
  每一方的发送和接收要2次，握手四次
  第二次的时候可以响应对方的发送能力 同时希望对方确认我们的发送能力
  SYN 同步序列号 一个随机的数字

- 四次挥手
  - 断开连接
  - A 发送 FIN 序列号 j
  - B 响应 ACK j + 1
  - B 发送 FIN 序列号 k
  - A 响应 ACK k + 1
