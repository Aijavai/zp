# 定位
- 文档流
    文档流是HTML 元素默认的布局方式，块级元素垂直排列，行内元素水平排列，遵循从上到下，从左到右的布局规则。
    - 脱离文档流
- position: relative
    - 相对定位，是相对于元素在文档流中的原来位置进行定位。
    - 不会离开文档流，原来在标准流的位置继续占有，后面的盒子继续按照标准流的位置排列。
    - display: none 
        - 隐藏元素，不占用空间，不会影响文档流布局。 不构建DOM树。
- position: absolute
    - 绝对定位，是相对于最近的非静态定位祖先元素进行定位。
    - 会离开文档流，原来在标准流的位置被释放，后面的盒子会占用原来的位置。
    opacity: 0.5 
    - 透明度，0 完全透明，1 完全不透明。

- position: fixed
    - 固定定位，是相对于浏览器窗口进行定位。
    - 固定定位的盒子不会随着浏览器的滚动而移动。
    - 会离开文档流，原来在标准流的位置被释放，后面的盒子会占用原来的位置。

- position: sticky
    - 粘性定位，是相对于浏览器窗口进行定位。
    - 粘性定位的盒子会在滚动到一定位置时固定在顶部。
    - 根据滚动的位置在relative和fixed之间切换。
      它的行为像relative定位，直到滚动到一定位置，才会固定在顶部。
    - 会离开文档流，原来在标准流的位置被释放，后面的盒子会占用原来的位置。

- position: static
    - 静态定位，是元素的默认定位方式。
    - 不会离开文档流，原来在标准流的位置继续占有，后面的盒子继续按照标准流的位置排列。
